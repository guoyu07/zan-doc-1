MSG_PEEK

MSG_PEEK

recv 方法的第二个参数，窥视socket缓存区中的数据。 设置MSG_PEEK参数后，recv读取数据不会修改指针，因此下一次调用recv仍然会从上一次的位置起返回数据。

None found

MSG_DONTWAIT

recv 方法的第二个参数，非阻塞接收数据，无论是否有数据都会立即返回

MSG_WAITALL

connect()

connect(string  $host, integer  $port, integer  $flag) : boolean

connect 连接到远程服务端，与setConnectTimeout结合使用，可支持连接超时

Parameters

string $host

远程服务的地址

说明：
    使用异步swoole_client时，host不能为域名，必须为ip
integer $port

远程服务的端口
integer $flag

[optional] 可选字段，默认为 0

     $flag 字段使用说明：
     UDP client：$flag = 1, 启用 udp connect，将绑定 $host 和 $port，数据发送使用send接口；
                 $flag = 0, 数据发送使用sendto
     TCP client：$flag = 1, 表示设置为非阻塞 socket;
                 $flag = 0, 使用阻塞socket，但是在异步模式下，$flag = 0无效。

Returns

boolean —

调用成功返回true，调用失败false

Tags

sleep()

sleep() : boolean

sleep 此方法会从事件循环中移除当前 socket 的可读监听，停止接收数据。 此方法仅停止从 socket 中接收数据，但不会移除可写事件，所以不会影响发送队列 sleep 操作与 wakeup 作用相反，使用 wakeup 方法可以重新监听可读事件

Returns

boolean —

成功返回 true，失败返回 false

Tags

since 3.1.0

wakeup()

wakeup() : boolean

wakeup 调用此方法会重新监听可读事件，将 socket 连接从睡眠中唤醒。 如果socket并未进入sleep模式，wakeup操作没有任何作用

Returns

boolean —

成功返回 true，失败返回 false

getpeername()

getpeername() : array|false

getpeername 获取对端 socket 的IP地址和端口，仅支持 SWOOLE_SOCK_UDP/SWOOLE_SOCK_UDP6 类型的 swoole_client 对象。 UDP协议通信客户端向一台服务器发送数据包后，可能并非由此服务器向客户端发送响应。可以使用此方法获取实际响应的服务器IP:PORT。
